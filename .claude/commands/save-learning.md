`/home/tohshima/project/atcoder/rust/notes/<問題番号>.md`

既存ファイルがあれば追記・更新する。なければ新規作成。

## ファイルのフォーマット

Obsidianで開くことを想定したMarkdownで書く。

### フロントマター（YAML）

```yaml
---
tags:
  - <学んだ概念のタグ（例: hashset, 計算量, イテレータ, rust基礎）>
problem: <問題番号>
date: <今日の日付 YYYY-MM-DD>
difficulty: <tessoku-A / tessoku-B / abc-X など>
---
```

### 本文の構成

1. **問題へのリンク**: `[[../../rust/tessoku-book/<問題番号>|問題文]]` 形式のObsidianウィキリンク
2. **問題の要約**: 1〜2行
3. **実装アプローチ**: 提出したコードと、もし改善案があればそれも
4. **このセッションで学んだこと**: 概念ごとにセクションを分けて記載

### 各概念セクションの書き方

- 概念の説明（なぜ必要か、何をするか）
- コード例（あれば）
- **知識レベル** を以下の3段階で記載:
  - `🟡 雰囲気理解` — 使い方はわかったが内部はまだ曖昧
  - `🟢 実装可能` — 自分で書ける
  - `🔵 説明可能` — 他人に説明できる

## 注意

- 解答ファイル（`src/bin/*.rs`）は編集しない
- ノートファイル（`rust/notes/*.md`）の作成・編集は問題なし
- 作成後は `git add` + `git commit` する（コミットメッセージ例: `docs: a03の学習ノートを追加`）
