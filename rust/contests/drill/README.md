# Drill — A 問題完答パターン練習

AtCoder の **A 問題で詰まらなくなる** ために、a-typeのよく出るパターンを詰め込んだドリル。  
a4 までやってみた感触（自力・写経）をもとに、**苦手分野を多めに**、**得意分野は確認程度に** 配分してある。

## 配分の意図（a1〜a4 のフィードバックより）

- ✅ できる: `input!`、`if/else`、`Vec` の読み込み、`.iter().min()/max()`
- ⚠️ あと一歩: 文字列スライス + parse（既出パターンの応用）
- 🟡 弱い: **`chars()` / `match` / イテレータ応用** / **エッジケース** / 出力フォーマット

→ 文字列を10問、`match` を4問、エッジケース系を3問。あとは桁操作・配列応用・偶奇/倍数/絶対値/階乗で全体を埋める。

## 問題一覧

### 既存（a1〜a4）
| 番号 | テーマ | キーワード |
|---|---|---|
| a1 | 合格判定 | if/else, `input!` 複数値 |
| a2 | バーコードチェック | `s[3..].parse()` |
| a3 | 最低気温 | `Vec`, `.iter().min()` |
| a4 | 母音カウント | `chars()`, `match` |

### 文字列操作（a5〜a14）★最重点強化
| 番号 | テーマ | キーワード |
|---|---|---|
| a5 | 文字数を数えよ | `.len()` / `chars().count()` |
| a6 | 先頭の文字 | `chars().next()` |
| a7 | 末尾の文字 | `chars().last()` |
| a8 | 部分文字列の取り出し | `&s[1..4]`、1-indexed 注意 |
| a9 | 'a' の個数 | `filter` + `count` |
| a10 | 大文字に変換 | `to_uppercase()` |
| a11 | 文字列の比較 | `==` |
| a12 | 文字列の連結 | `format!` |
| a13 | 文字を数値に | `c as i64 - '0' as i64` |
| a14 | 同じ文字の数 | `char` を input、`filter` |

### match 文（a15〜a18）★弱点強化
| 番号 | テーマ | キーワード |
|---|---|---|
| a15 | 月から季節 | `match` の範囲 `3..=5` |

| a17 | 文字の種類判定 | `match` の `char`、`|` で複数 |
| a18 | じゃんけんの勝者 | `match` でタプル、ガード |

### エッジケース注意（a19〜a21）★弱点強化
| 番号 | テーマ | キーワード |
|---|---|---|

### 桁操作（a22〜a24）
| 番号 | テーマ | キーワード |

### 出力フォーマット・計算（a25〜a27）
| 番号 | テーマ | キーワード |

| a25 | 西暦から世紀 | 切り上げ除算 |

### 配列応用（a28〜a30）
| 番号 | テーマ | キーワード |
|---|---|---|

## 取り組み方の提案

- **1日3〜5問** を目安に、テーマをまたいで解くと飽きない
- **詰まったら md のヒントを先に見る** → それでもダメなら写経 OK（A-4 のスタイルでよい）
- 解き終わったら **時間を空けて再挑戦**。再挑戦で何も見ずに通せたらマスター
- 苦手と感じたテーマは `notes/beginner/` に学びをまとめる

## 使い方

`src/bin/aN.rs` には `use proconio::input;` と空の `main()` だけのテンプレが用意されている。  
`cct aN` でサンプルテスト実行（例: `cct a5`）。

## ファイル構成

```
drill/
├── a1.md ... a34.md         # 問題文
├── README.md                # このファイル
├── b1.md ... b5.md          # B 問題系
├── src/bin/
│   ├── a1.rs ... a4.rs      # 既存の解答
│   ├── a5.rs ... a34.rs     # 空のテンプレ
│   └── b1.rs ... b5.rs
└── testcases/
    └── a1.yml ... a34.yml   # サンプルテストケース
```
