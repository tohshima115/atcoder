# B-15: 和が K になるペア（高速版）

配点: 300点

### 問題文

$N$ 個の整数 $a_1, a_2, \ldots, a_N$ が与えられる。  
$1 \le i < j \le N$ で $a_i + a_j = K$ となる組が **存在するか** 判定し、するなら `Yes`、しないなら `No` を出力せよ。

### 制約

- $2 \le N \le 2 \times 10^5$
- $1 \le K \le 2 \times 10^9$
- $1 \le a_i \le 10^9$
- 入力は全て整数

### 入力

```
N K
a_1 a_2 ... a_N
```

### 出力

条件を満たす組が存在するなら `Yes`、しないなら `No` を出力せよ。

---

### 入力例1

```
5 7
1 2 3 4 5
```

### 出力例1

```
Yes
```

$3 + 4 = 7$。

---

### 入力例2

```
3 100
1 2 3
```

### 出力例2

```
No
```

---

### 入力例3

```
2 4
2 2
```

### 出力例3

```
Yes
```

$2 + 2 = 4$（$i \ne j$ の別々の要素）。

---

### 練習ポイント

[[drill-b6]] と同じ問題だが **制約が $N \le 2 \times 10^5$**。二重ループ $O(N^2) = 4 \times 10^{10}$ は **TLE**（→ [[../../notes/reference/complexity-estimation|計算量の壁]]）。

`HashSet`/`HashMap` を使い、各要素 $a_j$ を見るとき「相方 $K - a_j$ が **これまでに出たか**」を O(1) で問い合わせれば $O(N)$。
**「引きながら更新する」オンライン処理**なので sort では逃げられない典型例。
