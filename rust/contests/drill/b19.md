# B-19: 和が K の区間の数

配点: 350点

### 問題文

$N$ 個の整数 $a_1, \ldots, a_N$ が与えられる。  
連続する部分列 $(a_l, a_{l+1}, \ldots, a_r)$ のうち、和がちょうど $K$ になるものの個数を求めよ。

### 制約

- $1 \le N \le 2 \times 10^5$
- $-10^9 \le a_i \le 10^9$（**負もありうる**）
- $-10^{14} \le K \le 10^{14}$
- 入力は全て整数

### 入力

```
N K
a_1 a_2 ... a_N
```

### 出力

和が $K$ になる連続部分列の個数を出力せよ。

---

### 入力例1

```
5 5
1 2 3 2 1
```

### 出力例1

```
2
```

`[2,3]` と `[3,2]` の 2 つ。

---

### 入力例2

```
5 0
1 -1 1 -1 1
```

### 出力例2

```
6
```

---

### 入力例3

```
3 10
1 2 3
```

### 出力例3

```
0
```

---

### 練習ポイント（C 頻出：累積和 ＋ seen-so-far）

二重ループで全区間 $O(N^2)$ は TLE。**累積和** $S$ を使うと、区間 $[l,r]$ の和 $= S_r - S_{l-1}$。
和が $K$ ⇔ $S_r - S_{l-1} = K$ ⇔ $S_{l-1} = S_r - K$。

各 $r$ について「$S_r - K$ に等しい累積和が**これまでに**何回出たか」を `HashMap` で数えれば $O(N)$。
[[drill-b15]] の「調べる→入れる」を累積和に応用した形。**`HashMap` を `{0:1}` で初期化**（空の prefix）するのがポイント。負があるので two-pointer は不可、これが Map 必須の典型。
