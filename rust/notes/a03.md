# A03 - Two Cards

問題: [[../../rust/tessoku-book/a03|A03問題文]]

## 問題の要約

赤カード N 枚・青カード N 枚から1枚ずつ選び、合計が K になるペアが存在するか判定する。

## 実装アプローチ

### O(N²) — 全探索（提出済み）

赤カードと青カードの全ペアを試す。N ≤ 100 なので十分間に合う。

```rust
for &vp in &p {
    for &vq in &q {
        if vp + vq == k { ... }
    }
}
```

### O(N) — HashSet を使う改善版

`q` を HashSet に入れておき、`k - vp` が存在するか O(1) で判定する。

```rust
use std::collections::HashSet;

let q_set: HashSet<i64> = q.iter().copied().collect();
for &vp in &p {
    if q_set.contains(&(k - vp)) {
        ans = true;
        break;
    }
}
```

| ステップ | 計算量 |
|---|---|
| HashSet 構築 | O(N) |
| p をループして判定 | O(N) × O(1) |
| 合計 | O(N) |

## このセッションで学んだこと

### HashSet

- 「ある値が含まれているか」を **O(1)** で判定できるデータ構造
- 配列の線形探索 O(N) との違いが競プロで効いてくる
- `.contains(&value)` で判定
- 重複排除にも使える

**知識レベル: 🟡 雰囲気理解**（使い方はわかった、内部構造はまだ）

---

### イテレータ・collect

- `q.iter()` → `&i64` の参照を返すイテレータ
- `.copied()` / `.cloned()` → 参照を値に変換（数値型は `.copied()` が正確）
- `.collect::<HashSet<i64>>()` → イテレータをコレクションにまとめる（型注釈が必要）

**知識レベル: 🟡 雰囲気理解**（パターンは使える、`&` と値の違いはまだ曖昧）

---

### 参照（`&`）と値

- `i64` は値そのもの、`&i64` は値のある場所を指すポインタ
- `iter()` が参照を返すのは所有権を移さないため
- 数値型（`Copy` トレイト）はコピーしても問題ないが、仕様として `&` が出てくる
- `String` や `Vec` などの大きなデータで参照のありがたみが増す

**知識レベル: 🟡 雰囲気理解**（所有権・借用は別途まとめて学ぶ予定）

---

### ラベル付き break

- `break` は直近のループしか抜けない
- 外側ループを抜けるには `'label:` をつけて `break 'label` を使う

```rust
'outer: for &vp in &p {
    for &vq in &q {
        if vp + vq == k {
            break 'outer;
        }
    }
}
```

**知識レベル: 🟢 実装可能**

---

### 型推論と型注釈

- `let mut ans = false;` → `bool` と推論される（注釈不要）
- `collect()` のように推論が迷う場面では型注釈 `:HashSet<i64>` が必要
- Rust の型名は `bool`（`boolean` はない）

**知識レベル: 🟢 実装可能**
