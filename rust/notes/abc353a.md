## 問題

[ABC353 A - Buildings](https://atcoder.jp/contests/abc353/tasks/abc353_a)

N 個のビルの高さが与えられる。1 番目のビルより高いビルの中で最も左のものの番号を出力する。なければ -1。

## 実装アプローチ

先頭要素 `h[0]` を基準に、2番目以降をループして最初に `h[i] > h[0]` となる `i` を探す。

```rust
use proconio::input;

fn main() {
    input! {
        n: usize,
        h: [i64; n]
    }
    let mut ans: i64 = -1;
    for i in 1..n {
        if h[i] > h[0] {
            ans = i as i64 + 1;
            break;
        }
    }
    println!("{}", ans);
}
```

## このセッションで学んだこと

### Rust の配列は 0-indexed

- `h[1]` は **2番目** の要素。1番目は `h[0]`
- 問題文が 1-indexed でも、コード上は 0-indexed で扱う
- 出力が 1-indexed なら `i as i64 + 1` のように変換が必要

**知識レベル**: 🟡 雰囲気理解（ミスが出やすいので注意）

---

### ループの範囲：`..` vs `..=`

| 書き方 | 意味 | 例 |
|---|---|---|
| `1..n` | 1 以上 n **未満** | 1, 2, ..., n-1 |
| `1..=n` | 1 以上 n **以下** | 1, 2, ..., n |

配列サイズ `n` の場合、有効インデックスは `0..n-1`。`..=n` にすると `h[n]` でパニック。

```rust
// NG: h[n] にアクセスしてパニック
for i in 1..=n { ... }

// OK
for i in 1..n { ... }
```

**知識レベル**: 🟢 実装可能

---

### 型キャストは `as`

`usize` を `i64` に変換するには `as` キーワードを使う。

```rust
let i: usize = 3;
let x: i64 = i as i64;
```

`i::i64` は**構文エラー**（`::` はモジュールパス区切り文字）。

**知識レベル**: 🟢 実装可能
