# Drill A-3 — 最低気温

[[../monday-drill|月曜ドリル]]の A-3。N 日間の気温の最小値を出力する。

## 自分の実装

```rust
use proconio::input;

fn main() {
    input! {
        n: usize,
        t: [i64; n]
    }
    let mut min: i64 = t[0];
    for &v in &t {
        if v < min {
            min = v;
        }
    }
    println!("{}", min)
}
```

正しく動く。最小値を求める基本パターン（初期値 = 最初の要素 → 残りと比較）。

## このセッションで学んだこと

### 問題文の `t_1〜t_n` と Rust の `t[0]〜t[n-1]`

問題文では 1-indexed（`t_1, t_2, ..., t_n`）、Rust の配列は 0-indexed。

| 問題文 | Rust |
|---|---|
| `t_1` | `t[0]` |
| `t_2` | `t[1]` |
| `t_n` | `t[n-1]` |

「入力は t1 から始まる」≠「Rust で t[1] から始まる」。
最小値の初期値を `t[0]` にするのは「**1番目の要素**を初期値にする」という意味で、ごく自然な定石。

- `t[1]` を初期値にしても**動く**（for で全要素を見るから）が、`n=1` のときに範囲外アクセスでパニックする
- 「とりあえず1個目を仮の答えにして、残りと比較する」という考え方が普遍的

知識レベル: 🟢 実装可能

### メソッドチェーン（イテレータパターン）

`.iter().min().unwrap()` のように `.` で繋いでいくのが **メソッドチェーン**。
Rust では超頻出のパターン。

```rust
let min = t.iter().min().unwrap();
```

仕組みは段階的に：

1. `t.iter()` — Vec から「**イテレータ**」（順番に要素を吐き出すもの）を作る
2. `.min()` — イテレータから最小値を探す → `Option<&i64>` を返す
3. `.unwrap()` — Option の中身を取り出して `&i64` に

よく使うイテレータメソッド：

| メソッド | 役割 |
|---|---|
| `.iter()` | Vec/配列をイテレータに |
| `.chars()` | String/&str を文字のイテレータに |
| `.min()` / `.max()` | 最小・最大（Option を返す） |
| `.sum()` / `.product()` | 合計・積（型注釈が必要なことが多い） |
| `.count()` | 個数 |
| `.enumerate()` | `(index, value)` のタプルに |
| `.rev()` | 逆順 |
| `.map(\|x\| ...)` | 各要素を変換 |
| `.filter(\|x\| ...)` | 条件を満たすものだけ残す |
| `.collect()` | Vec などに集める（型注釈が必要なことが多い） |

知識レベル: 🟡 雰囲気理解（書ける場面が増えてきた）

### `unwrap()` とは — `Option` / `Result` の中身を取り出す

Rust では「**失敗するかもしれない処理**」の結果が箱に包まれて返ってくる。

#### `Option<T>` — 中身があるか or ないか

```rust
enum Option<T> {
    Some(T),  // 中身あり
    None,     // 中身なし
}
```

例: `.iter().min()` は配列が空だと最小値がないので `Option<&T>` を返す。

#### `Result<T, E>` — 成功 or 失敗

```rust
enum Result<T, E> {
    Ok(T),    // 成功
    Err(E),   // 失敗
}
```

例: `.parse()` は文字列が数値じゃないと失敗するので `Result<T, ParseIntError>` を返す。

#### `.unwrap()` の動き

「**中身が絶対あると信じて取り出す**」メソッド。中身がなければパニックして落ちる。

```rust
let v = vec![3, 1, 4];
let min = v.iter().min();          // Option<&i32>
let min = v.iter().min().unwrap(); // &i32（中身を取り出した）

let s = "42";
let n: i64 = s.parse().unwrap();   // 失敗ありえないなら unwrap
```

競プロでは制約で「N ≥ 1」「入力は数値」が保証されてれば `unwrap()` で OK。
本番のサービス開発では基本使わない（パニック＝アプリ落ちるので）。

似たやつ:
- `.expect("メッセージ")` — パニック時にメッセージが出る。デバッグ用。
- `.unwrap_or(デフォルト値)` — 中身がなければデフォルト値を使う。

### 競プロでよく書く min/max の1行版

今回のコードは1行で書ける：

```rust
let min = t.iter().min().unwrap();
println!("{}", min);
```

ループで書くのは「処理を理解するため」には◎。慣れたら1行版にしてよい。
`.max()` も同じ書き方。
