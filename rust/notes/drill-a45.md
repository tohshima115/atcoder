## 問題

[[../../contests/drill/a45|問題文]]

N 個の品物を1箱に最大 K 個ずつ詰めるとき、必要な箱の最小個数を出力する。本質は **切り上げ除算**。

## 実装アプローチ

```rust
use proconio::input;

fn main() {
    input! {
        n: i64,
        k: i64,
    }
    let ans: i64 = (n + k - 1) / k;
    println!("{}", ans);
}
```

または:
```rust
let ans = n.div_ceil(k);   // Rust 1.73+ で標準利用可
```

最初に書いた `n / k + 1` は **割り切れるとき過剰カウントするバグ**。例: N=9, K=3 → 9/3+1 = 4 だが正解は 3。

---

## 学んだこと

### 切り上げ除算の定番イディオム

```rust
let ans = (n + k - 1) / k;
```

- 「割る数より1小さい数」を分子に足してから割る
- 割り切れるとき: `(9 + 2) / 3 = 11/3 = 3` ✓
- 余りあるとき: `(10 + 2) / 3 = 12/3 = 4` ✓
- 「あと1箱必要」な時だけ繰り上がる仕掛け

**他言語（C++/Python/Java...）でも同じ書き方が通用する**ので一生もの。

知識レベル: 🔵 説明可能

---

### `n / k + 1` がダメな理由

`n / k + 1` は **常に +1 する**。
- 余りがある時は +1 で正しい
- **割り切れる時にも +1 してしまう** → 過剰

「割り切れる時だけ +1 しない」は分岐が要るので素直に書くと:
```rust
if n % k == 0 { n / k } else { n / k + 1 }
```
これでもいいが、`(n + k - 1) / k` のほうが分岐なしで済む。

知識レベル: 🔵 説明可能

---

### `n.div_ceil(k)` — Rust 標準ライブラリの切り上げ

```rust
let ans = n.div_ceil(k);
```

- Rust 1.73 で安定化
- AtCoder の Rust バージョンで使えるなら一番きれい
- 内部的には `(n + k - 1) / k` 相当だがオーバーフロー対策込み

知識レベル: 🟢 実装可能

---

### 制約 $10^9$ を見たら反射的に `i64`

| 型 | 最大値（だいたい） |
|---|---|
| `i32` | $2.1 \times 10^9$ |
| `i64` | $9.2 \times 10^{18}$ |
| `usize` (64bit) | $1.8 \times 10^{19}$ |

- 制約 $1 \le N, K \le 10^9$
- `n + k - 1` を計算すると最大 $2 \times 10^9$ 近くなり **`i32` だと溢れる可能性**
- 入力受け取りから `i64` にしておけば計算中もずっと安全

**ルール**: 問題文で $10^9$ を見たら `i64`。$10^{18}$ までは `i64` で足りる。それ以上なら `i128` を考える。

知識レベル: 🔵 説明可能

---

### 切り上げ系の応用パターン

| やりたいこと | 書き方 |
|---|---|
| 切り上げ除算 | `(n + k - 1) / k` |
| 切り下げ除算 | `n / k` （整数除算のデフォルト） |
| 四捨五入 | `(n + k / 2) / k` （正の数限定） |
| 余り | `n % k` |

「N 個を K ずつのグループに分ける、最後の余りも1グループ」系問題で頻出。

知識レベル: 🟡 雰囲気理解
