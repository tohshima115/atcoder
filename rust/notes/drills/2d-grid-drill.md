# 2次元グリッド 克服ドリル

> 苦手の正体は「2次元を触った回数が足りない」だけ。**回数を踏んで手癖にする**ための自作問題集。
> 各問に使う機構（①〜⑤）は [[2d-grid-and-prefix]] と対応。辞書を引きながら解いてOK。
> 解答は載せない。**出力例で自己採点**（AtCoderのコードテスト or `cargo run` に入力を手貼り）。詰まったら壁打ち。

推奨順：G → S → T → P（やさしい→競プロ頻出）。1問ずつでいい。

---

## カテゴリ1：単純グリッド操作（①作る・走査・アクセス）

### G1 ★ セルの値を引く（1-index → 0-index 変換）
H×W の整数グリッド。Q 個のクエリ (r, c)（**1始まり**）について `grid[r][c]` を出力。
**使う機構：①／添字の -1**
```
入力
3 3
1 2 3
4 5 6
7 8 9
2
1 1
3 2
```
```
出力
1
8
```

### G2 ★ 各行の和・各列の和
H×W 整数グリッド。各行の和を H 行出力 → 続けて各列の和を空白区切り1行で。
**使う機構：①／二重ループの回し方（行優先 vs 列優先）**
```
入力
2 3
1 2 3
4 5 6
```
```
出力
6
15
5 7 9
```

### G3 ★★ 時計回りに90度回転
H×W の文字グリッドを時計回り90°回転して出力（結果は W×H）。
**使う機構：①／新旧の添字対応 `new[i][j] = old[H-1-j][i]`**
```
入力
2 3
abc
def
```
```
出力
da
eb
fc
```

### G4 ★★ 上下左右4近傍の和（境界処理）
H×W 整数グリッド。各セルについて、上下左右の隣（存在するものだけ）の合計を、同じ形で出力。
**使う機構：①／境界チェック（`i>0` `i+1<h` …）**
```
入力
2 2
1 2
3 4
```
```
出力
5 5
5 5
```

## カテゴリ2：累積和（④2次元累積和・⑤縦累積和）

### S1 ★★ 矩形和クエリ
H×W 整数グリッド。Q 個のクエリ (r1,c1,r2,c2)（1始まり）の矩形和を出力。
**使う機構：④**
```
入力
3 3
1 2 3
4 5 6
7 8 9
2
1 1 2 2
2 2 3 3
```
```
出力
12
28
```

### S2 ★★ 固定サイズ k×k の最大和
H×W 整数グリッド、整数 k。すべての k×k 正方形の和の最大値。
**使う機構：④で各窓を O(1)**
```
入力
3 3 2
1 2 3
4 5 6
7 8 9
```
```
出力
28
```

### S3 ★★★ 和がちょうど K の長方形の個数（= abc461 D の整数版）
H×W の **0/1 整数**グリッド、整数 K。和が K の長方形の個数。
**使う機構：⑤（行ペア固定 → 1次元に潰す → 和=K を数える [[drill-b15]]）**
※ abc461 D から「文字列→グリッド変換」を抜いた版。これが解ければ D は変換を足すだけ。
```
入力
3 4 3
1 0 0 1
1 1 0 1
0 1 1 0
```
```
出力
8
```

## カテゴリ3：文字列 → 2次元グリッド変換（②／③）

### T1 ★ '#' の個数
H×W の `#`/`.` グリッド。`#` の総数。
**使う機構：②または③／`== b'#'` の比較**
```
入力
3 4
#..#
.##.
#..#
```
```
出力
6
```

### T2 ★ 数字文字列グリッドの総和
H 行の数字文字列（各文字 0〜9）。全マスの数字の合計。
**使う機構：②（`b - b'0'`）**
```
入力
2 3
123
456
```
```
出力
21
```

### T3 ★★ 各行で最初に '#' が出る列（1始まり、無ければ -1）
**使う機構：③／行ごとに走査して break**
```
入力
3 4
..#.
....
#...
```
```
出力
3
-1
1
```

### T4 ★★★ 文字グリッドを 0/1 化して矩形和クエリ
`#`→1, `.`→0 に変換し、2次元累積和。Q 個の矩形 (r1,c1,r2,c2) 内の `#` の数を出力。
**使う機構：②変換 ＋ ④累積和の合わせ技**
```
入力
3 3
#.#
.#.
#.#
2
1 1 3 3
1 1 2 2
```
```
出力
5
2
```

## カテゴリ4：競プロ頻出の2次元グリッド

### P1 ★★★ 2次元いもす法（矩形に区間加算）
H×W グリッド（最初は全0）。Q 個の操作 (r1,c1,r2,c2) で矩形領域に +1。全操作後のグリッドを出力。
**使う機構：いもす（4隅に +1/-1 を置いて最後に2次元累積和）**
```
入力
3 3
2
1 1 2 2
2 2 3 3
```
```
出力
1 1 0
1 2 1
0 1 1
```

### P2 ★★★ 隣り合う '#' の組の数
H×W の `#`/`.` グリッド。横または縦に隣接する (`#`,`#`) の組の数。
**使う機構：①／隣接アクセス（右・下だけ見れば重複しない）**
```
入力
2 3
##.
.##
```
```
出力
3
```

### P3 ★★★★（発展・来週の道具）迷路の最短歩数（BFS）
H×W グリッド。`.` は通行可、`#` は壁、`S` 開始、`G` 目標。上下左右に1歩ずつ動くとき S→G の最短歩数。到達不能なら -1。
**使う機構：グリッド + dx/dy配列 + 2次元 visited + `VecDeque`（BFS）。**
※ BFS はまだ未習。`[[graph-basics]]` を起こすときの実戦台にする。今は「こういう形」を眺めるだけでOK。
```
入力
4 5
S....
.###.
...#.
.#..G
```
```
出力
7
```

## 進め方
- 1問解くごとに、使った機構を [[2d-grid-and-prefix]] で**引いてから**書く（辞書を引く癖）
- 全問で計算量も一応 GO/NO-GO（[[complexity-estimation]]）
- S3 が解けたら abc461 D は射程内（変換②を足すだけ）
- 詰まった問題は番号を言って壁打ち。解答が欲しければ出す（写経も可）
