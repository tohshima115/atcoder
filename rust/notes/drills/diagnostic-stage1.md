# 苦手分野診断 — Stage 1（広く浅くスキャン）

ABC の B〜易C で頻出する **8 つの型** を 1 問ずつ。すべて配点 200 点・標準 B 難易度。
目的は「どの型が苦手か」を炙り出すこと。**今週末土曜の ABC で AB 完答＋運が良ければ C** が目標。

## 進め方

1. b6 〜 b13 を順に解く（`cct b6` でテスト）。
2. 各問題を解いたら、下の表に **自己評価** を記入:
   - 🟢 **楽勝** — 詰まらず AC
   - 🟡 **解けたが時間/苦労** — 方針は立ったが実装やデバッグで手間取った
   - 🔴 **解けない** — 方針が立たない / 時間内に AC できない
3. 全部終わったら、🟡🔴 が付いた分野を **Stage 2 で深掘り**（難 B〜易 C を各 2〜3 問）。
   全部 🟢 なら Stage 2 は最小限でよい。

## 各型が ABC でどう出るか（なぜこの 8 つか）

- **二重ループ全探索(b6)**: B の定番。「条件を満たす組」を数える。型(オーバーフロー)の罠も。
- **文字列走査(b7)**: B 頻出。`chars`/`bytes`/`windows` のどれを使うかの引き出し。
- **シミュレーション(b8)**: 問題文どおりに実装する丁寧さ。クランプ・入力パースの地力。
- **累積和(b9)**: 「素直に解くと O(NQ) で TLE → 前処理で O(1)」の典型。易 C への橋。
- **約数列挙(b10)**: 「全部試すと間に合わない → √N」という計算量の気づき。
- **ソート＋貪欲(b11)**: 「どう並べると最適か」を見抜く。B〜C の貪欲の入口。
- **HashMap 集計(b12)**: 値が大きく配列添字が使えない → map。集計系 B の核。
- **bit 全探索(b13)**: N≤20 で 2^N 全探索。C でよく出る典型テク。
