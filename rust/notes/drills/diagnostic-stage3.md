## それぞれの「C で効く理由」

- **b19 累積和＋map**: 「区間の和が K」は prefix sum で $S_r-S_{l-1}=K$ に変形 → seen-so-far で O(N)。[[../problems/drill/drill-b15]] の発展。負があると two-pointer 不可で map 必須。**累積和は C 最頻出**。
- **b20 O(N²) 見抜き**: 制約 $N\le3000$ を見て「全区間 $O(N^2)$ でいい」と判断する型。難しい手法を探さず素直に。max−min が単調なので break も効く。
- **b21 DP**: 貪欲で解けない → 漸化式 $dp[i]=\max(dp[i-1], dp[i-2]+a_i)$。**DP の入口**。C で DP は超頻出。
- **b22 座標圧縮**: $10^9$ の値を「1,2,3…」に圧縮して配列添字を可能にする前処理。sort+dedup+binary_search または値→順位の map。
