## 問題

[[../../../contests/drill/a20|問題文]]

隣り合う要素の差 `|A[i+1] - A[i]|` の最大値を求める。N=1 のとき 0。

## 実装

```rust
use proconio::input;

fn main() {
    input! {
        n: usize,
        a: [i64; n]
    }
    let mut max: i64 = 0;
    for i in 0..(a.len() - 1) {
        let diff = (a[i] - a[i + 1]).abs();
        if diff > max {
            max = diff
        }
    }
    println!("{}", max);
}
```

**最初の失敗**: 絶対値の取り方がわからず「二乗して最大を求めてから平方根を取る」迂回路を実装。`sqer()` という存在しない関数を呼んでコンパイルエラー。

## 学んだこと

### `.abs()` — i64 に直接使える絶対値メソッド

`i64` などの数値型は `.abs()` を持っており、メソッドチェーンで直接使える。

```rust
let diff = (a[i] - a[i+1]).abs();  // これだけ
// ↑ 「二乗→最大→平方根」は不要
```

知識レベル: 🟢 実装可能

### n=1 のエッジケースは `0..0` で自然に処理される

`n=1` のとき `0..(n-1)` = `0..0` は空のレンジ。ループは一度も回らず初期値 0 がそのまま出力される。
わざわざ `if n == 1 { 0 }` と分岐を書く必要はない。

知識レベル: 🟢 実装可能

### 競プロでよく使う「一発解決」メソッド早見表

「数学的迂回路」を取る前に、まずメソッドがないか確認する習慣をつける。

| やりたいこと | 使うメソッド | `.iter()` 不要 |
|---|---|---|
| 絶対値 | `x.abs()` | ✅ 値に直接 |
| 2値の大きい方 | `a.max(b)` | ✅ 値に直接 |
| 2値の小さい方 | `a.min(b)` | ✅ 値に直接 |
| 範囲内にクランプ | `x.clamp(lo, hi)` | ✅ 値に直接 |
| 累積和 | `v.iter().sum::<i64>()` | `.iter()` 必要 |
| 最大値（配列全体） | `v.iter().max()` | `.iter()` 必要 |
| 条件で絞り込み | `v.iter().filter(\|&&x\| x > 0)` | `.iter()` 必要 |
| 各要素を変換 | `v.iter().map(\|&x\| x * 2)` | `.iter()` 必要 |

#### `.iter()` が必要な理由

`.iter()` は `Vec<T>` を **「一個ずつ取り出せる状態（イテレータ）」に変換する** メソッド。
`.sum()` や `.map()` はイテレータプロトコルを前提としているため、先に `.iter()` で変換が必要。

一方 `.abs()` / `.max(b)` / `.min(b)` は `i64` などの **値そのものに生えているメソッド** なので `.iter()` は不要。

### メソッドチェーンの `()` に引数を入れるパターン

`()` は関数呼び出しの記法なので、メソッドによって引数の種類は様々。

```rust
// 値を引数に取る
v.iter().take(3)          // 先頭3個だけ取る
v.iter().skip(2)          // 先頭2個をスキップ
v.iter().zip(other.iter()) // 2つのイテレータを並走させる

// クロージャ（ラムダ）を引数に取る
v.iter().map(|x| x * 2)         // 各要素を変換
v.iter().filter(|&&x| x > 0)    // 条件で絞り込み
v.iter().fold(0, |acc, &x| acc + x) // 集約（初期値, 関数）

// 引数なし
v.iter().max()
v.iter().sum::<i64>()     // 型パラメータで返り値の型を指定
v.iter().enumerate()      // (index, value) のペアにする
```

**クロージャ** (`|x| ...`) は「その場で定義する小さな関数」。`map` や `filter` はこれを受け取って各要素に適用する。
