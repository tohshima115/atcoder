## 問題

[[../../../contests/drill/a22|問題文]]

非負整数 N の10進数での桁数を出力する。N=0 のとき 1。

## 実装

```rust
use proconio::input;

fn main() {
    input! {
        n: i64,
    }
    let mut ans: i64 = 1;
    for i in (0..18).rev() {
        if n / 10_i64.pow(i as u32) >= 1 {
            ans += i;
            break;
        }
    }
    println!("{}", ans);
}
```

`ans = 1` で初期化し `ans += i`（= `i + 1`）で桁数を求める。N=0 のときはループが全部偽で初期値 1 のまま出力 → 正しい。

**最初の失敗**: `revers()`（スペルミス）、`10.dat(i)`（存在しない関数）、`` `[i64; 1]` ``（配列型を比較値に使おうとした）。

## 学んだこと

### 整数リテラルの型注釈 `10_i64`

Rustはリテラルに型を直接書ける。`_` は読みやすさのための区切りで省略可能。

```rust
10_i64   // i64 型の 10
10i64    // 同じ（アンダースコアなし）
10 as i64  // キャストでも同じ結果
```

競プロでは基本 `i64` を使うので、型が合わないと言われたら `10_i64` の形で解決することが多い。

知識レベル: 🟢 実装可能

---

### `.pow()` — 整数の累乗

```rust
10_i64.pow(3)    // 10^3 = 1000（整数）
2_f64.powi(3)    // 2^3 = 8.0（f64、整数乗）
2_f64.powf(0.5)  // 2^0.5 ≈ 1.41（f64、小数乗）
```

- 引数の型は `u32`（負の乗数は整数では意味がないため）
- `i64` の変数を渡すときは `x as u32` のキャストが必要
- Rustに `**` 演算子はない（Python/JSの記法）

知識レベル: 🟢 実装可能

---

### `1 << i` — 2の累乗のショートカット

`<<` はビットシフト演算子。「1を左にiビットずらす」操作。

```
1 << 0  = 1   (2^0)
1 << 1  = 2   (2^1)
1 << 2  = 4   (2^2)
1 << 3  = 8   (2^3)
```

2進数で「左に1桁ずらす = 2倍」なので `1 << i` が `2^i` と等価。`1` から始めるのは「2進数の1をずらすから」で、`2 << i` にすると `2^(i+1)` になってしまう。

**2の累乗限定のショートカット**。他の底には使えない。

知識レベル: 🟡 雰囲気理解

---

### 整数除算は切り捨て（小数にならない）

Rustの整数同士の `/` は小数点以下切り捨て。

```rust
123_i64 / 100  // → 1（1.23 ではない）
7_i64 / 2      // → 3（3.5 ではない）
```

小数が必要なときは `f64` にキャストしてから割る。

```rust
123_f64 / 100.0  // → 1.23
```

知識レベル: 🔵 説明可能

### 競プロで使う数値型まとめ

| 型 | 使う場面 |
|---|---|
| `i64` | 計算全般（デフォルト） |
| `usize` | 配列インデックス、`.len()` の戻り値 |
| `u32` | `.pow()` の引数など（求められたら使う） |
| `f64` | 小数が必要な計算 |
