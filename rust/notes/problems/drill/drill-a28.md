## 問題

[[../../../contests/drill/a28|問題文]]

奇数個の整数を昇順ソートして中央値を出力する。

## 実装

```rust
use proconio::input;

fn main() {
    input! {
        n: usize,
        mut a: [i64; n],
    }
    a.sort();
    let mid: f64 = if n % 2 == 0 {
        (a[n / 2 - 1] as f64 + a[n / 2] as f64) / 2.0
    } else {
        a[(n - 1) / 2] as f64
    };
    println!("{}", mid);
}
```

**最初の失敗**: `mut` を忘れてソートできなかった。インデックスを `(n+1)/2` にしてずれた（n=5で3番目のはずが4番目になった）。

---

## 学んだこと

### `a.sort()` — 昇順ソート

```rust
input! {
    n: usize,
    mut a: [i64; n],  // ← mut が必要（配列を書き換えるため）
}
a.sort();  // 昇順
```

- `sort()` は配列を直接書き換えるので `mut` が必要
- `f64` には `sort()` が使えない（`NaN` の存在で大小比較が保証できないため）
- 降順は `a.sort_by(|x, y| y.cmp(x))`

知識レベル: 🟢 実装可能

---

### 中央値のインデックス（0-indexed）

`n` が奇数のとき真ん中のインデックスは `n / 2`（= `(n-1) / 2` と同じ）。

```
n=1 → インデックス 0
n=5 → インデックス 2（1 1 [3] 4 5）
n=7 → インデックス 3（-2 -1 0 [3] 5 5 9）
```

`(n+1)/2` にするとインデックスが1つずれる。

知識レベル: 🟢 実装可能

---

### f64 の演算は全部 f64 で揃える

Rustは型が違う演算を自動変換しない。`f64` の式には `2.0` が必要。

```rust
(3.0_f64 + 4.0) / 2    // コンパイルエラー（f64とi32は混ざらない）
(3.0_f64 + 4.0) / 2.0  // OK → 3.5
```

知識レベル: 🔵 説明可能

---

### 最初から `[f64; n]` で受け取る選択肢

`as f64` を省けるが、`f64` は `sort()` が使えないので整数計算が混じる問題には向かない。「整数で受け取って必要なときだけ `as f64`」の方が安全。

知識レベル: 🟡 雰囲気理解
