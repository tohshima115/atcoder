# Drill A-4 — 母音カウント

[[../monday-drill|月曜ドリル]]の A-4。英小文字のみからなる文字列 `S` に含まれる母音（a, e, i, o, u）の個数を出力する。

## 自分の実装

```rust
use proconio::input;

fn main() {
    input! {
        s: String,
    }
    let mut ans: i64 = 0;
    for c in s.chars(){
        match c {
            'a' | 'e' | 'i' | 'o' | 'u'
            => ans += 1,
            _ =>{}
        }
    }
    println!("{}", ans);
}
```

`for c in s.chars()` で1文字ずつ取り出して、`match` で母音判定するパターン。

## このセッションで学んだこと

### 文字列を1文字ずつ処理する: `s.chars()`

`String` から1文字ずつ取り出すには `.chars()` を使ってイテレータにする。

```rust
for c in s.chars() {  // c は char
    // ...
}
```

`s.vec!` は無効（`vec!` は配列を作るマクロであってメソッドではない）。
**競プロで文字列を扱うときの9割はこの形で済む**。

知識レベル: 🟢 実装可能

### `Vec<char>` への変換

「インデックスでアクセスしたい」「2回ループしたい」など特殊な場合だけ `Vec<char>` にする。

```rust
let sep: Vec<char> = s.chars().collect();
```

ポイント：
- `.collect()` は「イテレータをコレクションに詰め直す」メソッド
- どの型に詰めるかを `.collect()` 単独では決められないので、左の型注釈 `: Vec<char>` で教える
- turbofish 記法でも書ける：`s.chars().collect::<Vec<char>>()`
- 同じ `.collect()` でも型注釈次第で結果が変わる：
  ```rust
  let v: Vec<char> = s.chars().collect();   // char の Vec
  let s2: String  = s.chars().collect();    // String
  ```

知識レベル: 🟡 雰囲気理解（必要になったときに使えればよい）

### `match` 構文 — 強化版 if-else

「ある値を、上から順にパターンと照合して、最初にマッチしたアームを実行する」構文。

```rust
match 調べたい値 {
    パターン1 => 式1,
    パターン2 => 式2,
    _        => 式3,   // それ以外全部
}
```

- `=>` は JS のアロー関数とは別物で `match` 専用記号。「マッチしたら右側を実行する」だけの意味
- 各アームは `,` で区切る
- 右側に複数行書きたいときは `{}` で囲む

```rust
match c {
    'a' | 'e' | 'i' | 'o' | 'u' => {
        ans += 1;
        println!("母音だった");
    }
    _ => {}
}
```

知識レベル: 🟡 雰囲気理解（構文として丸暗記でOK）

---

### `|` はパターンの「または」（`||` とは別物）

```rust
'a' | 'e' | 'i' | 'o' | 'u' => ...
```

`match` のパターン内でだけ使える「またはこのパターン」の意味。論理 OR `||` とは違う構文。

普通の `if` で書くなら：

```rust
if c == 'a' || c == 'e' || c == 'i' || c == 'o' || c == 'u' { ... }
```

`match` のほうが「値を場合分けしてる」意図が明確で読みやすい。

知識レベル: 🟢 実装可能

---

### ワイルドカード `_` と網羅性チェック

`match` は **網羅的（exhaustive）** でないとコンパイルエラーになる。「ありえる全パターンをカバーしろ」とコンパイラに要求される。

```rust
match c {
    'a' | 'e' | 'i' | 'o' | 'u' => ans += 1,
    // ❌ 他の文字をカバーしてないのでエラー
}
```

`_`（ワイルドカード）で「残り全部」を表してフタをする：

```rust
_ => {}    // ✅ 何もしないけど全パターンをカバー
```

知識レベル: 🟢 実装可能

---

### `match` は数値や範囲でも使える

文字列限定ではない。`i64`, `usize`, タプル、enum など何でもいける。

```rust
match x {
    0 => println!("zero"),
    1 => println!("one"),
    _ => println!("other"),
}
```

範囲は `..=`（以下）か `..`（未満）で書ける：

```rust
match score {
    0..=59   => println!("不合格"),
    60..=79  => println!("普通"),
    80..=100 => println!("優秀"),
    _        => println!("範囲外"),
}
```

知識レベル: 🟡 雰囲気理解

---

### マッチガード — `>=` のような条件は `if` で追加する

素のパターンには `>=` `<` などの比較は書けない。**マッチガード** という仕組みで `if` を後ろに付けて条件を追加する。

```rust
match x {
    n if n >= 100 => println!("100以上"),
    n if n >= 10  => println!("10以上99以下"),
    n if n < 0    => println!("負"),
    _             => println!("その他"),
}
```

`n` は「x の値をそのまま `n` という名前に束縛する」というパターン。そこに `if 条件` を追加できる。

ただし正直、こういう比較は普通に `if-else` で書いたほうが読みやすい：

```rust
if x >= 100 { ... }
else if x >= 10 { ... }
else if x < 0 { ... }
else { ... }
```

知識レベル: 🟡 雰囲気理解

---

### `match` vs `if-else` の使い分け目安

| ケース | おすすめ |
|---|---|
| 「a または b または c」みたいな**等値の列挙** | `match` + `\|` |
| **範囲**で区切りたい（`0..=59` みたいに連続） | `match` + `..=` |
| `>=` `<` などの**比較演算** | `if-else` |
| `Option` / `Result` / enum の**中身の分岐** | `match`（後で出てくる）|

「`match` は等値の場合分けが得意」と覚えておけば十分。
