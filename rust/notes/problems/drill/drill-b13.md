## 問題

[[../../../contests/drill/b13|問題文]]

$N$ 個の整数から 1 個以上を選び、合計をちょうど $K$ にできるか判定（`Yes`/`No`）。$N \le 20$。

## 実装アプローチ

**手法は知っていたが「書き方」で詰まった**回。$N \le 20$ なので $2^N$ 通りの選び方を全部試す = **bit 全探索**。

```rust
use proconio::input;

fn main() {
    input! {
        n: usize,
        k: i64,
        a: [i64; n],
    }
    let mut ok = false;
    for mask in 0u64..(1 << n) {          // 2^n 通りの「選び方」
        let mut sum = 0i64;
        for i in 0..n {
            if (mask >> i) & 1 == 1 {     // i ビット目が立ってたら a[i] を選ぶ
                sum += a[i];
            }
        }
        if sum == k { ok = true; }
    }
    println!("{}", if ok { "Yes" } else { "No" });
}
```

> 詰まったのは「$2^N$ 通りを試す」というロジックではなく、**それを Rust の bit 操作でどう表現するか**だった。

---

## 学んだこと

### bit 全探索の核：部分集合 = N ビットの整数

「N 個から選ぶ/選ばない」の組合せは $2^N$ 通り。これを **0 〜 $2^N-1$ の整数 `mask`** で表す。
`mask` の各ビットが「その要素を選んだか」を表す。例：`mask = 0b101` → 0 番目と 2 番目を選んだ。

```rust
for mask in 0u64..(1 << n) {  // 1 << n は 2^n
    for i in 0..n {
        if (mask >> i) & 1 == 1 { /* a[i] を選んだ */ }
    }
}
```

`mask` は**ごく普通の整数**（`0,1,2,…`）。整数はもともと内部で 2 進数なので、ビット演算でその桁を「選択フラグ」として読む、という発想。

知識レベル: 🟡 雰囲気理解（→ 詳細は [[bit-operations]]）

### `(mask >> i) & 1 == 1` の分解（丸暗記でいいが仕組みも）

「`mask` の `i` ビット目が立ってるか」を調べるイディオム。2 ステップ:

1. **`mask >> i`** … i 桁ぶん右にずらし、調べたいビットを右端(0桁目)へ寄せる
2. **`& 1`** … `1`(=`0b1`)との AND で 0 桁目以外を消す → 結果は `0` か `1`

```
mask=0b1011, i=2 →  0b1011>>2 = 0b10  →  0b10 & 1 = 0  （2桁目は0）
mask=0b1011, i=0 →  0b1011>>0 = 0b1011 → 0b1011 & 1 = 1 （0桁目は1）
```

Rust では `&` が `==` より結合が強いので `(mask >> i) & 1 == 1` は `((mask>>i)&1)==1` と正しく解釈される（C の有名な罠は Rust では起きない）。読みやすさで `(mask >> i)` のカッコは付ける。

知識レベル: 🟢 実装可能

### `0u64` vs `0i64`（mask の型）

この問題（$N \le 20$）なら **`0i64` でも `0u64` でも OK**。`1 << 20` ≈ 100万でどちらにも余裕で収まる。

`u64`（符号なし）を選ぶ理由は「N が大きいときの保険」:
- mask は「数」でなく**ビットの集合**なので符号なしが意味的に自然
- `1i64 << 63` は符号ビットが立って**負の数**になり範囲が壊れる → N が 31/63 に近いと `u64` 必須

| N の上限 | mask の型 |
|---|---|
| N ≤ 20〜30 | `i64`/`u64`/`usize` どれでも可 |
| N が 31, 63 に近い | **`u64`**（符号ビットまで使う） |

競プロの bit 全探索は $N \le 20$ 前後が大半（$2^{20} \approx 10^6$ が現実的上限）。実戦では型に神経質にならなくてよいが「ビット列は符号なし」の習慣は保険になる。

知識レベル: 🟢 実装可能

## このセッションの総括

| 学び | 要点 |
|---|---|
| bit 全探索 | 部分集合を $0..2^N$ の整数 `mask` で表し、`(mask>>i)&1` で各要素の選択を読む |
| `0b` 記法 | `0b1011 == 11`。型でなく「2進で書くだけ」。整数は元々内部で2進数 |
| mask の型 | $N \le 20$ なら何でも可。大きいなら `u64` |

**学び**: 「アルゴリズムは知ってるのに解けない」= 手法でなく **Rust での書き方（イディオム）が無い** パターン。bit 全探索は典型イディオムを 1 度写経すれば次から書ける。
