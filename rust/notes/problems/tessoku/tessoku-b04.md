## 問題

[[../../../contests/tessoku-book/b04|問題文]]

2進数の文字列 N を10進数に変換して出力する。

## 実装アプローチ

```rust
use proconio::input;

fn main() {
    input! {
        n: String,
    }
    let mut ans: i64 = 0;
    for (i, c) in n.chars().enumerate(){
        if c == '1' {
            ans += 1 << (n.len() - 1 - i)
        }
    }
    println!("{}", ans);
}
```

各文字を左から走査し、`'1'` のとき `1 << (n.len() - 1 - i)` で桁の重み（2のべき乗）を加算する。

## このセッションで学んだこと

### `enumerate()` — インデックスと値を同時に取る

イテレータに `.enumerate()` を付けると `(index, value)` のタプルが返る。
`for (i, c) in ...` のように分解して受け取る。

```rust
for (i, c) in "1101".chars().enumerate() {
    // (0, '1'), (1, '1'), (2, '0'), (3, '1')
}
```

順序は **常に `(index, value)`**。「番号をつける = 番号が先」と覚える。

**知識レベル**: 🟡 雰囲気理解 — 何度か書いてようやく定着しそう

---

### タプルのパターンマッチング（分解）

`for` の変数部分に `(i, c)` と書くのはタプルをその場で分解する書き方。

```rust
let pair = (0, '1');
let (i, c) = pair;  // i = 0, c = '1'
```

`for (i, c) in ...` はこれをループのたびに行っているイメージ。

**知識レベル**: 🟡 雰囲気理解

---

### `.iter()` vs `.chars()`

| | 対象 | 返す型 |
|---|---|---|
| `.iter()` | `Vec` などのコレクション | 要素への参照 |
| `.chars()` | `String` / `&str` | `char` |

`String` に `.iter()` は使えない。文字を1文字ずつ取り出すときは必ず `.chars()`。

**知識レベル**: 🟡 雰囲気理解 — 使い分けの理由はまだ曖昧

---

### 初期値の書き方（型による違い）

| 型 | 初期値 |
|---|---|
| `i64`, `usize` など数値 | `0` |
| `String` | `String::new()` または `"".to_string()` |
| `Vec<T>` | `Vec::new()` または `vec![]` |

数値はリテラルで直接書けるが、`String` や `Vec` はヒープにメモリを確保するオブジェクトなので `new()` が必要。

**知識レベル**: 🟡 雰囲気理解

---

### ビット演算 `1 << k`

`1 << k` は 2^k を表す。2進数の各桁の重みを計算するのに使える。

左から `i` 番目のビットの重みは `1 << (n.len() - 1 - i)`（右から数えた位置に変換）。

**知識レベル**: 🟢 実装可能
