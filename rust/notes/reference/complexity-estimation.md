# 計算量をパッと見積もるコツ（チートシート）

> ABC で問題を読んだら **最初に制約欄の N を見て、許される計算量を逆引きする**。
> N² を毎回計算するのではなく、「N の大きさ → 計算量」の対応表を丸暗記するのがプロのやり方。

## 大前提：基準は「10⁸」

- **AtCoder のマシンは 1 秒で約 10⁸（1億）回**の計算ができる。
- 実行時間制限はたいてい 2 秒。安全を見て **操作回数 10⁸ までなら間に合う** を基準にする。
  - 10⁷ … 余裕 / 10⁸ … セーフ圏 / 10⁹ … ギリ〜アウト寄り / 10¹⁰ … TLE

⚠️ 比較相手は `10^18` ではない。`N=10⁵` で N²=10¹⁰ は「10¹⁸ より小さい」が **TLE**。
速度の壁は **10⁸**。

## 逆引き表（これを丸暗記する）

| N の上限 | 間に合う計算量 | 典型解法 |
|---|---|---|
| N ≤ 10〜11 | O(N!) | 順列全探索（next_permutation） |
| N ≤ 20〜25 | O(2ᴺ) | bit 全探索（drill-b13） |
| N ≤ 500 | O(N³) | 三重ループ、ワーシャルフロイド |
| N ≤ 5000 | O(N²) | 二重ループ（drill-b6） |
| N ≤ 10⁵〜10⁶ | O(N log N) | ソート・二分探索・累積和（drill-b9） |
| N ≤ 10⁷〜10⁸ | O(N) | 一重ループ |
| N ≥ 10⁹ | O(log N), O(1) | 数学・二分探索 |

**使い方は逆引き**：「N² が間に合う？」と考えるのではなく、
**N を見る → 表で許される計算量を引く → その範囲で一番ラクな解法を選ぶ**。

例）b6 は N=2000 → 表の「N≤5000（O(N²)）」の段に入る → 二重ループで OK。
N² を実際に計算する必要すらない。

## ラクする計算法：指数だけで考える

数を全部計算せず、**10 の肩（指数）の足し算・掛け算**で概算する。

- N = 2000 ≈ 10³·³
- N² → 指数を 2 倍 → 10⁶·⁶ ≈ 4×10⁶ → 基準 10⁸ より下 → OK
- N = 10⁵ → N² = 10¹⁰ → 基準 10⁸ 超え → 二重ループ不可、log を落とせ

「N が 10ᵏ なら N² は 10²ᵏ、N³ は 10³ᵏ」と指数の掛け算で一瞬。

## 「速度の壁」と「型の壁」は別物（混同注意）

3 つの大きな数を**別の用途**として覚える：

| 数 | 何の壁か | 意味 |
|---|---|---|
| **10⁸** | 速度の壁 | 1 秒で回せる操作回数の上限（計算量の判断） |
| **10⁹** | i32 の壁 | `i32` は約 ±2.1×10⁹ まで。超えるなら i64/usize |
| **10¹⁸** | i64 の壁 | `i64`/`usize` は約 ±9.2×10¹⁸ まで |

- 「N² が間に合うか」＝ **速度の壁（10⁸）** の話。
- 「a_i + a_j がオーバーフローするか」＝ **型の壁（10⁹/10¹⁸）** の話。
- b6 で `a_i+a_j` を i64 にしたのは型の壁、N² が OK かは速度の壁。**この 2 つを切り分ける**。

→ 詳しくは [[integer-types-overflow]]。

## 本番の手順（毎問これをやる）

1. 問題を読んだら **まず制約欄の N を見る**
2. 逆引き表で「許される計算量」を引く（指数だけで概算）
3. その計算量で解ける手法を選ぶ
4. 別途「値の最大」を見て **型（i32 で足りるか i64 か）** を決める

20〜30 問やると「N=2000 を見た瞬間に二重ループが見える」感覚になる。
