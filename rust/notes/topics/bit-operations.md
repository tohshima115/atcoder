# ビット演算と 2 進数（`& | ^ ! << >>`・`0b` 記法・bit全探索）

> `>>` や `&` には「**2 進数の**」という枕詞が勝手に付く、と読み替えると腑に落ちる。
> 整数はもともと内部で 2 進数で記憶されている。ビット演算はその桁を直接いじる道具。

## `0b` は「2 進で書く」記法（型ではない）

```rust
let a = 11;       // 10進で書いた 11
let b = 0b1011;   // 2進で書いた 11   ← a と同じ値！
let c = 0xB;      // 16進で書いた 11  ← これも同じ
// a == b == c
```

`0b`/`0x` は**書き方の指定**で、できる数は普通の整数。`0b1011 == 11`。
ソースでビットの並びを見せたいときに使うだけ。

## ビット演算子の一覧

例は `0b1100`（=12）と `0b1010`（=10）:

| 記号 | 演算 | 意味 | 結果 |
|---|---|---|---|
| `&` | AND | 両方 1 なら 1 | `0b1000` (=8) |
| `\|` | OR | どちらか 1 なら 1 | `0b1110` (=14) |
| `^` | XOR | **違う**とき 1（同じなら 0） | `0b0110` (=6) |
| `!` | NOT | 全ビット反転 | `!0b1100` |
| `<<` | 左シフト | 左へずらす（×2ⁿ） | `0b1100<<1 = 0b11000` |
| `>>` | 右シフト | 右へずらす（÷2ⁿ） | `0b1100>>1 = 0b0110` |

注意:
- **NOT は `!`**（C 系の `~` ではない。Rust は bool 否定と同じ記号を使い回す）
- 代入版あり: `^=`, `&=`, `|=`, `<<=`, `>>=`（例 `a ^= b`）

## XOR の便利な性質（競プロ頻出）

- `a ^ a == 0` … 同じ数を 2 回 XOR すると消える
- `a ^ 0 == a`
- → 「ペアになってない 1 個だけ見つける」系で、全部 XOR すると答えが残る定番テク

知識レベル: 🟡 雰囲気理解

## bit 全探索のイディオム（[[drill-b13]]）

$N \le 20$ 程度なら $2^N$ 通りの「選ぶ/選ばない」を全部試せる。部分集合を整数 `mask` で表す:

```rust
for mask in 0u64..(1 << n) {          // 2^n 通り（1<<n = 2^n）
    for i in 0..n {
        if (mask >> i) & 1 == 1 {     // i ビット目が立ってたら i 番目を選択
            // a[i] を使う
        }
    }
}
```

- **`(mask >> i) & 1`**: 「i ビット目を右端へ寄せて、右端だけ残す」→ 0 か 1。立ってるか判定の常套句
- Rust は `&` が `==` より強いので `(mask>>i)&1==1` は正しく `((mask>>i)&1)==1`（C の罠なし）
- **mask の型**: $N\le20$ なら `i64`/`u64`/`usize` どれでも可。N が 31/63 に近いなら `1<<63` が符号ビットを立てるので `u64` 必須

## `&`/`|` はビットと論理で別物（混同注意）

| 記号 | 役割 | 例 |
|---|---|---|
| `&`（1個・値の間） | ビット AND | `mask & 1` |
| `&&`（2個） | 論理 AND（かつ・bool用） | `a>0 && b>0` |
| `\|`（1個・値の間） | ビット OR | `a \| b` |
| `\|\|`（2個） | 論理 OR（または・bool用） | `a>0 \|\| b>0` |

記号の文脈読み（位置で意味が決まる）の全体像は [[references-deref]] を参照。
