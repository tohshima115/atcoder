# 型選び・オーバーフロー・整数除算

> **繰り返し課題**: `i32`/`i64`/`usize`/`f64` のどれで受けるか、`as` でいつ変換するか、整数除算で小数が消える、を毎回考え直している。
> 出てきた問題: [[drill-a22]] [[drill-a26]] [[drill-a28]] [[drill-a45]] [[drill-a49]]

## 型の使い分け（競プロのデフォルト）

| 型 | 最大値（だいたい） | 使う場面 |
|---|---|---|
| `i64` | $9.2\times10^{18}$ | 計算全般（**まずこれ**） |
| `usize` | $1.8\times10^{19}$ | 配列インデックス・`.len()` |
| `u32` | $4.3\times10^{9}$ | `.pow()` の引数など求められたら |
| `f64` | — | 小数が必要なとき |
| `i32` | $2.1\times10^{9}$ | **競プロでは基本使わない**（溢れやすい） |

## 🔑 ルール: $10^9$ を見たら反射で `i64`

- 制約に $10^9$ → `i32`（最大 $2.1\times10^9$）だと **合計や中間計算で溢れる**
- 入力受け取りから `i64` にしておけば計算中ずっと安全（[[drill-a45]]）
- 「合計を取る瞬間」にもう一度型を意識する: $N\times10^9 = 10^{11}$ は `i32` で溢れる（[[drill-a49]]）
- $10^{18}$ までは `i64`、それ以上は `i128`

## 整数除算は切り捨て → 小数は `as f64`

```rust
7 / 2          // → 3（3.5 にならない）
sum as f64 / n as f64    // 平均など小数が要るとき（[[drill-a26]]）
```

f64 の式は **全部 f64 で揃える**。`(3.0 + 4.0) / 2` はエラー、`/ 2.0` が要る（[[drill-a28]]）。
小数フォーマットは `println!("{:.2}", x)`。

## 切り上げ除算（分岐なし）

```rust
(n + k - 1) / k     // 一生もの。他言語でも通用（[[drill-a45]]）
n.div_ceil(k)       // Rust 1.73+ ならこれ
```

`n / k + 1` は **割り切れる時に過剰カウント** するバグ（典型ミス）。

## リテラルの型注釈と pow

```rust
10_i64.pow(3)    // 整数累乗。引数は u32 → 変数なら x as u32（[[drill-a22]]）
2_f64.powf(0.5)  // 小数乗・ルート（f64）
1 << i           // 2^i のショートカット（ビットシフト）
```

## f64 が使えない場面

`f64` は `NaN` のせいで `sort()` できない（[[drill-a28]]）。整数計算が混じるなら「整数で受けて必要な時だけ `as f64`」が安全。
