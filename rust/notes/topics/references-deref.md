# 参照と参照外し（&・*・copied・クロージャの &&）

> **繰り返し課題**: `filter` の `|&&x|`、`map` の `|&x|`、`max().unwrap()` の後の `*`、`.copied()`…参照がいつ何枚付くか分からず「呪文」で乗り切っている。
> 出てきた問題: [[drill-a21]] [[drill-a30]] [[drill-a35]] [[drill-a41]] [[drill-a43]] [[drill-a54]]

> 競プロで扱うのはほぼ `Copy` 型（`i64`,`char`,`usize`…）。**最初は呪文で丸暗記でOK**、ここはRustで一番難しい所。

## 記号の文脈読み（位置で意味が決まる）

Rust の `&` `*` `|` は**置かれた場所で意味が切り替わる**。「役割が多くて大変」の正体はこれ。見分けは位置だけ:

| 記号 | 役割 | 例 |
|---|---|---|
| `&`（値と値の間） | **ビット AND** | `mask & 1` |
| `&`（値の前） | **参照を作る**（借りる） | `&x`, `&vec` |
| `&`（パターンの中） | **参照を剥がす** | `for &x in v`, `\|&x\|` |
| `&`（型の前） | **参照型** | `&i64` |
| `&&`（2個） | **論理 AND**（かつ・bool用） | `a>0 && b>0` |
| `*`（値の前） | **deref**（参照を剥がす） | `*op == 'U'` |
| `*`（値と値の間） | **掛け算** | `a * b` |
| `\|`（値と値の間） | **ビット OR** | `a \| b` |
| `\|...\|`（2本で囲む） | **クロージャの引数** | `.map(\|x\| x+1)` |
| `\|`（match の中） | パターンの区切り（または） | `1 \| 2 => ...` |
| `\|\|`（2個） | **論理 OR**（または・bool用） | `a>0 \|\| b>0` |

**見分けのコツ**:
- **値と値の間**の `&` `\|` `*` → 計算（ビット演算・掛け算）
- **値の前**の `&` `*` → 参照を作る／剥がす
- **2個重なった** `&&` `\|\|` → 論理（bool 専用）
- **`\| \|` で囲む** → クロージャの引数

例: `mask >> i & 1`（`&` が値の間 → ビット演算）、`*op == 'U'`（`*` が値の前 → deref）。
`*` と `&`-in-pattern は表裏一体: `*` は式の中で剥がす、`&` はパターンの中で剥がす（同じことを違う場所で）。

```rust
// 同じ「参照を剥がす」を 2 ルートで
for (op, x) in &ops { if *op == 'U' { pos += *x; } }   // 式で * 剥がし
for &(op, x) in &ops { if op == 'U' { pos += x; } }     // パターンで & 剥がし
```

→ ビット演算側の詳細は [[bit-operations]]。

## 3種のイテレータ — 参照 vs 値

| メソッド | 要素の型 | 元 |
|---|---|---|
| `vec.iter()` | `&T`（参照） | 借りる |
| `vec.into_iter()` | `T`（値） | 消費する |
| `s.chars()` | `char`（値） | 生成系（借りるが値で返す） |

`chars()` は UTF-8 をデコードして `char` を**生成**するので最初から値 → `.copied()` 不要で `collect::<String>()` できる（[[drill-a41]]）。`iter()` は借りるだけなので `&T`、値が要る場面で外す必要がある。

## クロージャの `&` 枚数

| 書き方 | 場面 | なぜ |
|---|---|---|
| `\|&&x\|` | `filter` / `find` | `iter()` の `&T` に filter 側がもう1枚 `&` を足す（[[drill-a30]] [[drill-a52]]） |
| `\|&x\|` | `map` で値として使う | 参照1枚を外して `x: T` |
| `\|x\|` | `map` で `.to_string()` 等を呼ぶ | auto-deref で参照のまま呼べる（[[drill-a35]]） |

```rust
.filter(|&&x| x >= k)   // = .filter(|x| **x >= k)
.map(|&x| x * 2)
.map(|x| x.to_string()) // 参照のまま OK
```

## `max().unwrap()` の後の `*`

```rust
*a.iter().max().unwrap()
// max() → Option<&i64> / unwrap() → &i64 / * → i64
```

`iter()` 由来なので参照、計算に使うには `*` で外す（[[drill-a21]]）。値が欲しいだけなら [[option-result-handling]] も参照。

## `.copied()` — `|&&x|` 地獄からの脱出

```rust
let rev: String = v.iter().rev().copied().collect();   // &char → char
let set: HashSet<i64> = a.iter().copied().collect();
// これらは全部同じ意味
.copied()   .map(|x| *x)   .map(|&x| x)
```

| 道具 | いつ |
|---|---|
| `.copied()` | 中身が `Copy`（`i64`,`char`,`bool`,`usize`…）← 競プロはほぼこれ |
| `.cloned()` | `Clone` だが `Copy` でない（`String`,`Vec`…） |

⚠️ `*` は **1個**の参照外し演算子。`*v.iter()...collect()` のようにイテレータ全体には効かない。要素ごとに `.map(|x| *x)` か `.copied()`（[[drill-a41]]）。

## `for` ループでの参照パターン（[[drill-a43]] [[drill-b8]]）

```rust
for (x, v) in queries  { ... }   // 消費。x: usize, v: i64（もう使わないなら最楽）
for &(x, v) in &queries { ... }  // 借りる＋外側の & を外して値。x: usize, v: i64
for (x, v) in &queries { ... }   // 借りる。マッチエルゴで x: &usize, v: &i64
for &v in &a { ... }             // 借りて値で受ける定番
```

`for ... in v` は暗黙に `v.into_iter()`（消費）。あとで使うなら `&v` で借りる。

### ⚠️ `&` は「タプルの外側」に 1 枚だけ（[[drill-b8]] で詰まった所）

`queries: Vec<(usize, i64)>` を `&queries` で回すと、各要素の型は:

```
&(usize, i64)       ← 参照は「タプル全体」の外側に 1 枚
(&usize, &i64) ではない！  ← 各フィールドに参照、ではない
```

だから剥がす `&` も**外側に 1 個**だけ置ける。中の各フィールドに `&` を配るのは NG:

```rust
for &(x, v) in &queries { ... }  // ✅ 外の & 1 枚を外の & で剥がす → x,v は値
for (&x, &v) in &queries { ... } // ❌ コンパイルエラー！
                                 //   error: expected `usize`, found `&_`
                                 //   フィールドには参照が無いので剥がせない
```

`(&x, &v)` が許されるのは、イテレータが `(&usize, &i64)`（参照のタプル）を返すとき
（例: 2 本の iter の `.zip()`）であって、`&Vec<(T,U)>` の `&(T,U)` ではない。

**覚え方**: `&vec` の `&` はタプルの外に 1 枚 → 剥がす `&` もパターンの外に 1 枚（`&(x, v)`）。

### マッチエルゴで受けた場合の `*` の要否（[[drill-b8]]）

`for (op, x) in &step`（`step: Vec<(char, i32)>`）→ マッチエルゴで `op: &char`, `x: &i32`。
両方とも参照だが、**演算によって `*` の要否が違う**:

| 演算 | `&T` のまま使える？ | 例 |
|---|---|---|
| `+` `-` `+=` `-=`（算術） | ✅ 不要（std が `&` 付き演算を実装） | `ans + x`, `ans += x` |
| `==` `<`（比較・値と値） | ❌ 要 `*` | `*op == 'U'`（or `op == &'U'`） |

→ `ans + x` は `ans + *x` と等価。算術は剥がさず比較だけ剥がす、で動く。
統一したいなら `for &(op, x) in &step` にすると全部値になり `*` が消える。
